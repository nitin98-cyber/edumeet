# PROJECT ABSTRACT

## EduMeet: Faculty Availability & Appointment System

### Overview
EduMeet is a web-based appointment management system designed to streamline the process of scheduling meetings between students and faculty members in educational institutions. The system eliminates the traditional manual approach of coordinating appointments through phone calls, emails, or physical visits.

### Purpose
The primary purpose of this system is to provide a digital platform where:
- Faculty members can define their available time slots
- Students can view faculty availability and book appointments
- Administrators can monitor and manage the entire appointment process

### Scope
The system covers three main user roles:
1. **Students**: Can register, login, view faculty availability, book appointments, and track appointment status
2. **Faculty**: Can login, set availability slots, view appointment requests, and approve/reject appointments
3. **Admin**: Can manage users and monitor all appointments in the system

### Key Features
- Secure user authentication for all user types
- Real-time availability checking
- Automatic conflict detection to prevent double booking
- Appointment status tracking (Pending, Approved, Rejected, Completed)
- User-friendly interface with responsive design
- Email notifications for appointment updates (optional)

### Technology Stack
- **Frontend**: HTML5, CSS3, JavaScript
- **Backend**: Python Flask Framework
- **Database**: MySQL
- **Development Model**: Waterfall SDLC

### Benefits
- Saves time for both students and faculty
- Reduces scheduling conflicts
- Provides transparent appointment tracking
- Maintains digital records of all appointments
- Improves communication efficiency

### Target Users
- Educational institutions (Colleges, Universities)
- Students seeking faculty guidance
- Faculty members managing student consultations
- Administrative staff overseeing appointment systems
